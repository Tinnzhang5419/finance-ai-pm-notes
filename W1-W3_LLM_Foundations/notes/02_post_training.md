## 背景：为什么需要 Post-training

Base Model 训练完之后：
- 参数（Attention权重）已全部确定，不再变化
- 具备丰富的世界知识
- **但只会"续写文本"，不会"好好回答问题"**

```
你问：  "如何做红烧肉？"
Base Model：  "如何做红烧肉？如何做糖醋排骨？如何做清蒸鱼？..."
              （续写模式，不是回答模式）
```

Post-training 的目标：把 Base Model 调教成**有帮助、无害、诚实**的对话助手。

---

## Post-training 三步

```
Step 1: SFT（监督微调）         ← 教模型"怎么回答"
Step 2: RM （训练奖励模型）      ← 教模型"什么是好回答"
Step 3: RL （强化学习优化）      ← 让模型持续朝好回答方向进化
```

---

## Step 1 · SFT（Supervised Fine-Tuning，监督微调）

### 做法

标注员写出高质量的"问题 + 示范回答"对，模型在此基础上继续训练：

```
问题：  "帮我写一封辞职信"
标注员示范回答：  "尊敬的领导，您好，兹定于……"

问题：  "解释一下什么是通货膨胀"
标注员示范回答：  "通货膨胀是指……"
```

### SFT vs 预训练对比

| | 预训练 | SFT |
|--|--|--|
| 数据量 | 万亿 token | 几万～几十万条对话 |
| 标注 | 无（自监督） | 有（标注员写示范） |
| 训练目标 | 预测下一个词 | 学会对话格式和风格 |
| 参数变化 | 从零开始 | 在 Base Model 基础上微调 |

### SFT 的局限

标注员的示范只告诉模型"这个回答是好的"，但无法区分"A回答比B回答更好"。  
→ 这就是需要第二步 RM 的原因。

## 幻觉（Hallucination）

### 什么是幻觉

模型自信地给出**听起来合理但实际错误或捏造**的回答。

```
你问：  "李明辉教授在哪所大学任教？"
模型答：  "李明辉教授现任北京大学教授，研究方向为……"
实际：  李明辉教授根本不存在
```

### ⚠️ 幻觉的根本原因：预训练阶段

幻觉不是 SFT 造成的，根源在预训练的训练目标：

> 模型学的是**"预测下一个最可能的 token"**，不是**"保证事实正确"**。  
> 模型天生没有"我不知道"这个选项，只有"下一个词最可能是什么"。

### SFT 与幻觉的关系

SFT **没有制造幻觉**，但改变了幻觉的表现形式，让它更危险：

```
Base Model 的幻觉：  跑偏、答非所问、续写乱飘
                    （容易被用户察觉）

SFT 之后的幻觉：    用流畅、自信、完整的对话格式编造
                    （更像真的，更难被察觉）
```

**准确说法：**
```
幻觉根源  →  预训练目标（预测token，不保证事实）
SFT 的作用 →  让幻觉表现得更自信、更流畅、更难识别
```

### 缓解幻觉的方法

- **RAG**（检索增强生成）：回答前先检索外部知识库，基于事实生成
- **工具调用**：接入搜索引擎，搜不到就说不知道
- **RLHF**：在训练样本中加入"拒绝回答"的示范，教模型说"我不确定"

### 面试口述版

> "幻觉的根本原因在预训练阶段——模型的训练目标是预测下一个token，而不是保证事实准确，所以它天生没有'我不知道'这个选项。SFT 没有制造幻觉，但让幻觉更难被发现，因为模型学会了用流畅自信的对话格式来编造内容。缓解方法主要有RAG、工具调用，以及在RLHF训练中加入拒绝回答的样本。"
