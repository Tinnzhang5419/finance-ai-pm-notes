# LLM Base Model：预训练数据处理流程

> 模块：W1–W3 LLM Foundations  
> 来源：个人学习笔记整理  
> 更新：2026-06-26

---

## 概览

Base Model（基础模型）的核心是在**海量文本**上做无监督预训练（Next Token Prediction）。  
数据质量直接决定模型质量——"Garbage in, garbage out"。  
因此在喂给模型之前，原始网页数据需要经过一套完整的**数据清洗流水线**。

```
原始网页 (Common Crawl)
    ↓
URL过滤
    ↓
文本提取
    ↓
语言过滤
    ↓
Gopher过滤（质量过滤）
    ↓
MinHash去重
    ↓
C4过滤
    ↓
自定义过滤
    ↓
PII去除
    ↓
干净的预训练语料
```

---

## 各步骤详解

### Step 1 · URL 过滤（URL Filtering）

**目的**：在下载前就排除低质量来源，节省计算资源。

- 过滤已知垃圾/色情/恶意域名黑名单
- 只保留信誉良好的域名（新闻、学术、百科等）
- 去除过短 URL、重定向链、参数垃圾页

**金融场景类比**：类似数据治理中的"数据源准入"——不合规数据源直接拦截在入口。

---

### Step 2 · 文本提取（Text Extraction）

**目的**：从 HTML 原始页面中提取干净的正文文本。

- 去除 HTML 标签、JS 代码、CSS 样式
- 提取 `<body>` 正文，丢弃导航栏、广告、页脚
- 工具：`trafilatura`、`jusText`、`BeautifulSoup`

**关键点**：同一个 URL 可能有大量模板化重复内容（如网站 header/footer），需识别并剔除。

---

### Step 3 · 语言过滤（Language Filtering）

**目的**：按目标语言比例筛选文档。

- 常见做法：只保留目标语言（如英语）占比 **≥65%** 的文档
- 工具：`fastText` 语言识别模型、`langdetect`
- 多语言模型则会按语言分桶，按比例采样

**思考点**：比例阈值是超参数，设太高会丢失有价值的双语内容；设太低引入噪声。

---

### Step 4 · Gopher 过滤（Gopher Filtering）

**目的**：基于启发式规则过滤低质量文档（DeepMind Gopher 论文提出）。

常见规则包括：
| 规则 | 说明 |
|------|------|
| 文档长度 | 过滤词数 < 50 或 > 100,000 的文档 |
| 平均词长 | 平均词长 < 3 或 > 10 个字符 |
| 符号比例 | `#` 等特殊符号占比过高 |
| 数字比例 | 数字占文本比例过高（可能是乱码/代码） |
| 重复行比例 | 高度重复行（可能是模板页） |
| 停用词覆盖 | 英语常见词（the/be/of）密度过低 |

**本质**：用廉价的统计指标快速识别并丢弃垃圾文本。

---

### Step 5 · MinHash 去重（MinHash Deduplication）

**目的**：大规模近似去重，删除语义相似或完全重复的文档。

- **精确去重**：URL 或文档 hash 完全一致 → 直接删除
- **模糊去重**：用 MinHash + LSH 找到「高度相似」的文档对 → 保留一份
- 意义：重复数据会让模型过拟合，且浪费算力

**类比**：类似数据库中的 `DISTINCT` + 相似度匹配，但规模是亿级文档。

---

### Step 6 · C4 过滤（C4 Filters）

**目的**：Google 在 C4 数据集中提出的一套补充清洗规则。

常见规则：
- 保留以标点符号结尾的句子（过滤不完整句）
- 去除包含脏话/屏蔽词的页面
- 去除代码页（包含 `{` `}` 的行超过阈值）
- 去除 "lorem ipsum" 占位文本
- 过滤 Wikipedia 引用格式噪声

**与 Gopher 的区别**：Gopher 偏向文档级统计，C4 偏向句子级规则。

---

### Step 7 · 自定义过滤（Custom Filters）

**目的**：各家模型根据自身需求添加的额外过滤逻辑。

常见做法：
- 用小型**分类器**判断文档质量（训练数据来自人工标注的高质量样本，如 Wikipedia、书籍）
- 过滤特定领域内容（法律风险、政治敏感）
- 比例控制：确保不同来源（网页/书籍/代码/论文）的混合比例符合设计目标

**代表案例**：LLaMA 用 fastText 分类器打分，只保留「Wikipedia 相似度高」的页面。

---

### Step 8 · PII 去除（PII Removal）

**目的**：删除个人身份信息，避免模型记忆并泄露隐私。

PII 类型包括：
| 类型 | 示例 |
|------|------|
| 姓名 + 联系方式 | 张三，138xxxx1234 |
| 身份证号 | 310xxxxxxxxxxxxxxx |
| 邮箱地址 | xxx@gmail.com |
| 家庭住址 | 上海市徐汇区xxx路xx号 |
| 银行卡/信用卡号 | 6225xxxxxxxx |
| IP 地址 | 192.168.x.x |

处理方式：
- **正则匹配**：结构化 PII（手机号、身份证、邮箱）
- **NER 模型**：命名实体识别，提取姓名、地址等非结构化 PII
- **替换策略**：用占位符 `[NAME]` `[PHONE]` 替换，而非直接删除整段

## 整体流程总结

```
数据量变化（示意）：
Common Crawl 原始数据   ~100%
↓ URL过滤               ~60%
↓ 文本提取              ~55%
↓ 语言过滤              ~40%
↓ Gopher过滤            ~25%
↓ MinHash去重           ~15%
↓ C4 + 自定义过滤       ~10%
↓ PII去除               ~10%（量不变，内容脱敏）
最终预训练语料           原始数据的 ~5–15%
```

> **核心洞察**：大模型的「智识」来自海量数据，但真正喂进去的只有原始抓取数据的一小部分。  
> 数据质量 > 数据数量——这也是为什么小而精的数据集（书籍、论文、高质量网页）权重往往比普通网页高。

## 面试口述版（30秒版本）

> "Base Model 的预训练数据要经过完整的清洗流水线：先从 URL 过滤掉低质量来源，再做 HTML 文本提取，然后按语言比例过滤；之后用 Gopher 的启发式规则做质量过滤，MinHash 做大规模去重，C4 规则做句子级清洗，各家再加自定义分类器；最后做 PII 脱敏，去掉身份证、手机号等隐私信息。整个流程下来，实际用于训练的数据大概只有原始爬取量的 5–15%。"
