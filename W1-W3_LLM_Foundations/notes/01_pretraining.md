# LLM Pre-training 完整笔记

> 模块：W1–W3 LLM Foundations  
> 更新：2026-06-26

---

## 总览：Base Model 是怎么炼成的

```
Step 1: 下载并预处理互联网数据
Step 2: Tokenization（词元化）
Step 3: Embedding + Attention（向量化 + 注意力）
Step 4: 神经网络训练（得到 Base Model）
```

---

## Step 1 · 下载并预处理互联网数据

原始数据主要来自 Common Crawl（对整个互联网的爬取快照）。  
但原始数据充满噪声，需要经过完整的清洗流水线：

```
原始网页
  ↓ URL过滤          只保留高质量来源域名
  ↓ 文本提取          去除HTML/JS/CSS，只留正文
  ↓ 语言过滤          保留目标语言占比≥65%的文档
  ↓ Gopher过滤        统计规则剔除低质量文档
  ↓ MinHash去重       近似算法删除高度相似文档
  ↓ C4过滤            句子级规则清洗
  ↓ 自定义过滤        各家模型自己的分类器
  ↓ PII去除           删除身份证/手机号等隐私信息
最终预训练语料（约为原始数据的5–15%）
```

**三层过滤的区别：**

| 过滤器 | 粒度 | 核心逻辑 |
|--------|------|----------|
| Gopher | 文档级 | 统计指标（词数、符号比例、重复行） |
| MinHash | 文档级 | 相似度去重，保留唯一内容 |
| C4 | 句子级 | 内容规则（标点结尾、无脏话、无占位文本） |

**核心洞察：** 数据质量 > 数据数量。最终用于训练的只有原始爬取量的 5–15%。

## Step 2 · Tokenization（词元化）

**目标：** 把文本切成模型能处理的最小单元（token），映射为数字 ID。

```
文本  →        token        →  数字ID
"tokenization"  →  ["token", "ization"]  →  [1234, 567]
```

**为什么要切成子词（subword）：**
- 整词切分：词表会无限大（每个词一个ID）
- 字母切分：序列太长，丢失词的语义
- 子词切分：词表可控，又能处理未见过的词

**Tokenization 只做机械切词，不处理语义。**  
同一个词无论什么含义，token ID 相同。语义的区分在下一步完成。

---

## Step 3 · Embedding + Attention（向量化 + 注意力）

### 3.1 Embedding：静态向量化

每个 token ID 映射到一个向量（一串数字），用来表示这个词的初始含义。

```
token ID → 向量
1234     → [0.2, -0.1, 0.5, ...]  （几百到几千维）
567      → [0.5,  0.3, 0.1, ...]
```

向量空间有语义算术特性：
```
king - man + woman ≈ queen
```

**⚠️ 关键纠正：**  
"bank"（银行）和"bank"（河岸）是**同一个 token ID，同一个初始向量**。  
Tokenization 和 Embedding 阶段不区分一词多义，区分靠 Attention。

**⚠️ 静态 Embedding 不是没用——它是 Attention 的输入原料**  
token ID 只是编号（如1234），没有数学意义，无法做运算。  
Attention 需要向量才能计算权重，所以静态 Embedding 是必经的第一步。

```
token ID → 静态Embedding → Attention计算 → 动态语境向量
  编号        原料              加工              成品
```

类比：静态 Embedding 是生鸡蛋，Attention 是烹饪过程，动态向量是最终的菜。  
同样的原料（生鸡蛋），不同语境（炒/煮/蒸）出不同结果。  
**静态 Embedding 是跳板，Attention 才是核心。**

---

### 3.2 Attention：动态融合上下文

**Attention 不是预测，是权重分配。**  
它回答的问题是：理解当前这个词时，应该重点参考句子里哪些词？

```
"动物太累了，因为它走了很长的路"

模型理解"它"时：
"动物" ← 权重 0.8  （高度相关）
"累了" ← 权重 0.1
"路"   ← 权重 0.1
```

通过 Attention，同一个"bank"在不同句子里会生成**不同的向量**：

```
"I went to the bank to deposit money"  →  bank的向量偏向"金融机构"
"I sat by the river bank"              →  bank的向量偏向"河岸"
```

这就是从静态 Embedding 升级到动态语义理解的关键。

**Attention 权重的时间线：**
```
训练阶段：权重参数不断调整
训练完成：权重参数固定不变
推理阶段：权重固定，但每次输入不同，注意力结果动态计算
```

类比：计算器的运算规则固定，但每次输入不同的数字，结果不同。

---

### 3.3 完整向量化链条

```
文本
  ↓ Tokenization（机械切词）
token ID（同型异义 = 同一个ID）
  ↓ Embedding（静态向量）
初始向量（每个token一个固定向量）
  ↓ Attention（动态融合上下文）
语境向量（现在"bank"的两个含义被区分了）
  ↓
进入神经网络预测
```

## Step 4 · 神经网络训练

### 4.1 神经网络是什么

神经网络是 Transformer 的底层概念，Transformer 是神经网络的一种架构。

```
神经网络（1980s，大概念）
  ├── RNN（顺序处理，容易遗忘远处的词）
  ├── CNN（擅长图像）
  └── Transformer（2017，并行处理，长距离依赖）← 现在主流
```

最简单的理解：
> 神经网络是一台有几十亿个旋钮的机器。训练就是不断转动旋钮，直到机器输出越来越接近正确答案。

### 4.2 Transformer 架构

```
Transformer（2017，完整 Encoder + Decoder）
  ├── 只取 Encoder → BERT（双向，擅长理解）
  └── 只取 Decoder → GPT （单向，擅长生成）← ChatGPT/Claude 这条路线
```

| | BERT | GPT |
|--|--|--|
| 使用部分 | Encoder | Decoder |
| 看上下文 | 双向（左右都看） | 单向（只看左边） |
| 训练任务 | 填空（预测被遮住的词） | 续写（预测下一个词） |
| 擅长 | 理解、分类、问答 | 生成、对话、续写 |

### 4.3 训练过程：自监督学习

**预训练完全无监督，不需要人工标注。**  
监督信号来自文本本身——答案就藏在原文里。

```
训练数据：  "今天天气很好"
输入：      "今天天气很___"
模型预测：  "好"
正确答案：  "好"（原文就是）
计算误差 → 反向传播 → 调整参数 → 重复千亿次
```

这叫 **Self-supervised Learning（自监督学习）**：  
看起来无监督，实际上监督信号藏在数据里。

这也是为什么互联网文本如此珍贵——几千亿句话 = 几千亿道自动出题的练习题。

## 全流程总结

```
互联网原始数据
  ↓ Step 1: 清洗（过滤/去重/脱敏）→ 干净语料
  ↓ Step 2: Tokenization → token ID序列
  ↓ Step 3: Embedding → 静态向量
            Attention  → 动态语境向量
  ↓ Step 4: 神经网络训练（自监督，预测下一词）
            反向传播调整参数，重复千亿次
  ↓
Base Model（知识丰富，但不会对话）
  ↓
Post-training / RLHF（下一章）
  ↓
能对话的大模型（ChatGPT / Claude）
```
